# Warum die Nutzung von GitHub Pages für unsere Website vertretbar ist

## Zweck & Funktionsweise

Wir hosten unsere statische Marketing-Website über **GitHub Pages**, einen Service der amerikanischen Firma **GitHub Inc.**. GitHub Pages dient dazu, Websites direkt aus einem GitHub-Repository heraus auszuliefern und gestaltet den technischen Betrieb sehr einfach, stabil und wartbar.

Die Nutzung ist insbesondere dann sinnvoll, wenn keine komplexen dynamischen Funktionen (z. B. Benutzerdatenbanken oder Login-Bereiche) erforderlich sind und die Seite primär für Informations- und Marketingzwecke dient.

## Vertretbare Gründe für die Nutzung

Wir nutzen GitHub Pages aus folgenden sachlichen Gründen:

- **Kosteneffizienz** – GitHub Pages erlaubt das Hosting einer statischen Website ohne erhebliche Hosting-Kosten.  
- **Technische Zuverlässigkeit** – Die Inhalte werden global ausgeliefert und GitHub sorgt für eine hohe Verfügbarkeit.  
- **Geringer Wartungsaufwand** – Änderungen am Inhalt können einfach über Versionskontrolle und Pull Requests erfolgen, was gerade für kleine Teams ohne dedizierte Infrastruktur vorteilhaft ist.

## Datenschutzrechtliche Einordnung

### Legitimation der Verarbeitung

Die Nutzung von GitHub Pages bedeutet technisch, dass beim Abruf der Website personenbezogene Daten (wie z. B. IP-Adresse oder Browserinformationen) an GitHub übertragen werden, da GitHub die Inhalte ausliefert. Auch wenn dies eine Übermittlung in die **USA** sein kann, wendet GitHub nach eigenen Angaben geeignete Rechtsrahmen wie das **EU-US Data Privacy Framework (DPF)** an, um ein angemessenes Datenschutzniveau zu gewährleisten.

Auf dieser Grundlage kann die Nutzung über Art. 6 Abs. 1 lit. f DSGVO legitimiert werden, wenn ein berechtigtes Interesse an der Bereitstellung der Website besteht und keine überwiegenden schutzwürdigen Interessen der Betroffenen entgegenstehen.

### Typische Nutzungsszenarien

Da GitHub Pages statisch ist und keine serverseitigen personenbezogenen Funktionen bietet, eignet sich der Dienst besonders für:

- **Marketingseiten und Unternehmenspräsentationen**
- **Dokumentationen und Portfolio-Seiten**
- **Blogs ohne Login- oder Nutzeranalysefunktionen**

Wie in vielen praktischen Beispielen beschrieben, wird dabei oft erklärt, dass keine Besucher- oder Nutzerdaten im klassischen Sinne gesammelt oder ausgewertet werden, sondern lediglich technisch nötige Daten zur Auslieferung der Inhalte übertragen werden.

## Fazit

Die Nutzung von GitHub Pages ist für einfache, statische Marketing-Websites **vertretbar**, wenn:

- die Zwecke der Verarbeitung klar sind und dokumentiert werden,
- eine transparente Datenschutzerklärung vorliegt,
- keine sensiblen personenbezogenen Daten über den statischen Service selbst verarbeitet werden.

Die genannten Gründe begründen ein berechtigtes Interesse im Sinne der DSGVO, das gegenüber möglichen Risiken abgewogen wurde.
